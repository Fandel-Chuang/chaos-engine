# ChaosEngine v0.1 并行任务计划

> **目标：** 将剩余 v0.1 工作拆解为 4 个独立子模块，由 subagent 并行执行

**当前状态：** Core/ECS/Log/Plugin/Network/Render 已完成，编译通过

---

## 任务拆解（4 路并行）

### Module A: Lua 运行时绑定
**负责：** Subagent A
**依赖：** public_api（已有）
**产出：**
- `src_c/script/ce_lua.h` — Lua VM 封装接口
- `src_c/script/ce_lua.c` — Lua 5.4 嵌入、C API 绑定
- `src_lua/shared/hello.lua` — 示例脚本
- 更新 CMake 链接 lua

### Module B: 单元测试
**负责：** Subagent B
**依赖：** 现有所有模块
**产出：**
- `tests/unit/test_math.c` — 数学库测试
- `tests/unit/test_memory.c` — 内存池测试
- `tests/unit/test_ecs.c` — ECS 测试
- `tests/unit/test_network.c` — 网络层测试
- `tests/CMakeLists.txt` — 测试构建

### Module C: 编辑器 UI 面板
**负责：** Subagent C
**依赖：** public_api
**产出：**
- `src_cpp/ui/editor_ui.h` — 编辑器面板接口
- `src_cpp/ui/editor_ui.cpp` — Dear ImGui 面板实现（层次结构、Inspector、控制台）
- `src_cpp/log_observer/log_observer.h/cpp` — 第八模式日志面板
- 更新 editor_main.cpp 集成 UI

### Module D: CI 规则 + 网络示例
**负责：** Subagent D
**依赖：** 无
**产出：**
- `ci/rules/check_cross_ref.py` — 跨目录引用检测脚本
- `ci/rules/check_cpp_in_c.py` — C++ 语法侵入检测
- `src_c/runtime/ce_server_main.c` — TCP 回显服务器示例
- `.github/workflows/build.yml` 或等效 CI 配置

---

## 执行顺序

```
Subagent A (Lua)     ─┐
Subagent B (Tests)   ─┼── 并行启动
Subagent C (Editor)  ─┤
Subagent D (CI)      ─┘
        │
        ▼
  主 Agent 汇总验证 → 编译 → 提交
```

## 上下文注入

每个 subagent 需要知道：
1. 项目目录结构
2. public_api 接口签名
3. 编译方式（cmake）
4. 代码风格（纯 C / C++17 分层）
